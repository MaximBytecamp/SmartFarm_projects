# Практическая работа по ООП

## SmartFarm — система управления умной фермой

---

## 1. Описание проекта

Необходимо разработать консольное Python-приложение **SmartFarm**, которое моделирует работу современной умной фермы.

Система должна позволять управлять:

* растениями;
* животными;
* теплицами;
* загонами;
* сотрудниками;
* датчиками;
* задачами;
* складом ресурсов;
* отчётами по состоянию фермы.

Проект направлен на закрепление основных принципов объектно-ориентированного программирования.

---

## 2. Цель работы

Закрепить на практике ключевые темы ООП в Python:

* классы и объекты;
* атрибуты и методы;
* инкапсуляция;
* защищённые и приватные атрибуты;
* свойства `@property`;
* наследование;
* полиморфизм;
* абстрактные классы;
* композиция;
* агрегация;
* магические методы;
* собственные исключения;
* разбиение проекта на модули.

---

## 3. Легенда

Агротехническая компания разрабатывает систему **SmartFarm** для управления современной фермой.

На ферме есть теплицы, загоны, растения, животные, сотрудники, датчики температуры и влажности, задачи по уходу и склад ресурсов.

Программа должна помогать отслеживать состояние фермы, назначать задачи сотрудникам, проверять показатели датчиков, ухаживать за растениями и животными, а также формировать итоговый отчёт.

---

## 4. Функциональные требования

Приложение должно уметь:

1. Создавать ферму.
2. Создавать теплицы и загоны.
3. Добавлять растения в теплицы.
4. Добавлять животных в загоны.
5. Создавать сотрудников разных типов.
6. Назначать задачи сотрудникам.
7. Отмечать задачи выполненными.
8. Создавать и использовать датчики.
9. Проверять состояние растений.
10. Проверять состояние животных.
11. Управлять складом ресурсов.
12. Формировать отчёт по ферме.
13. Обрабатывать ошибки через собственные исключения.

## 6. Основные классы

### 6.1. `FarmObject`

Абстрактный базовый класс для объектов фермы.

От него должны наследоваться классы:

* `Plant`;
* `Animal`.

#### Атрибуты:

* `_id` — защищённый идентификатор;
* `_name` — защищённое название объекта.

#### Методы:

* `get_info()` — абстрактный метод;
* `id` — свойство для получения идентификатора;
* `name` — свойство для получения названия;
* `__str__()` — строковое представление объекта.

---

### 6.2. `Plant`

Класс растения.

Наследуется от `FarmObject`.

#### Атрибуты:

* `_id`;
* `_name`;
* `plant_type`;
* `growth_stage`;
* `health_level`;
* `water_required`.

#### Методы:

* `water()` — полить растение;
* `grow()` — перевести растение на следующий этап роста;
* `check_health()` — проверить здоровье растения;
* `get_info()` — получить информацию о растении;
* `__str__()` — строковое представление;
* `__lt__()` — сравнение растений по уровню здоровья.

#### Этапы роста:

```python
growth_stages = [
    "семя",
    "росток",
    "растение",
    "цветение",
    "плодоношение"
]
```

#### Условия:

Если здоровье растения меньше `0` или больше `100`, необходимо вызвать собственное исключение.

---

### 6.3. `Animal`

Класс животного.

Наследуется от `FarmObject`.

#### Атрибуты:

* `_id`;
* `_name`;
* `animal_type`;
* `age`;
* `health_level`;
* `is_hungry`.

#### Методы:

* `feed()` — покормить животное;
* `heal()` — вылечить животное;
* `check_health()` — проверить здоровье;
* `get_info()` — получить информацию о животном;
* `__str__()` — строковое представление.

#### Условия:

Если животное голодное, метод `check_health()` должен сообщать, что животное требует кормления.

Если здоровье животного ниже `40`, программа должна рекомендовать вызвать ветеринара.

---

* `Agronomist`;
* `Veterinarian`.

#### Атрибуты:

* `_id`;
* `_name`;
* `position`;
* `tasks`.

#### Методы:

* `assign_task(task)` — назначить задачу;
* `complete_task(task)` — выполнить задачу;
* `work()` — абстрактный метод;
* `get_info()` — получить информацию о сотруднике;
* `__len__()` — количество задач у сотрудника.

### 6.9. `TemperatureSensor`

Класс датчика температуры.

#### Логика метода `check_status()`:

| Температура     | Состояние     |
| --------------- | ------------- |
| меньше 15°C     | холодно       |
| от 15°C до 30°C | нормально     |
| больше 30°C     | слишком жарко |

---

### 6.10. `HumiditySensor`

Класс датчика влажности.

#### Логика метода `check_status()`:

| Влажность     | Состояние      |
| ------------- | -------------- |
| меньше 30%    | слишком сухо   |
| от 30% до 70% | нормально      |
| больше 70%    | слишком влажно |

## 10. Пример демонстрационного кода

```python
from models.farm import Farm
from models.greenhouse import Greenhouse
from models.animal_zone import AnimalZone
from models.plant import Plant
from models.animal import Animal
from models.employee import Agronomist, Veterinarian
from models.task import Task
from models.sensor import TemperatureSensor, HumiditySensor


farm = Farm("SmartFarm")

greenhouse = Greenhouse(1, "Теплица №1")
animal_zone = AnimalZone(2, "Загон №1")

temp_sensor = TemperatureSensor(1, "Датчик температуры", 24, "°C")
humidity_sensor = HumiditySensor(2, "Датчик влажности", 62, "%")

greenhouse.add_sensor(temp_sensor)
greenhouse.add_sensor(humidity_sensor)

tomato = Plant(1, "Томат", "овощ", "росток", 85, True)
cucumber = Plant(2, "Огурец", "овощ", "растение", 76, True)

greenhouse.add_plant(tomato)
greenhouse.add_plant(cucumber)

cow = Animal(1, "Белка", "корова", 4, 90, False)
goat = Animal(2, "Стрелка", "коза", 2, 70, True)

animal_zone.add_animal(cow)
animal_zone.add_animal(goat)

agronomist = Agronomist(1, "Иван Петров", "агроном")
veterinarian = Veterinarian(2, "Мария Смирнова", "ветеринар")

task1 = Task(
    "Полить томаты",
    "Проверить влажность и полить растения",
    "средний"
)

task2 = Task(
    "Проверить корову",
    "Осмотреть состояние здоровья животного",
    "высокий"
)

agronomist.assign_task(task1)
veterinarian.assign_task(task2)

farm.add_zone(greenhouse)
farm.add_zone(animal_zone)
farm.add_employee(agronomist)
farm.add_employee(veterinarian)
farm.add_task(task1)
farm.add_task(task2)

farm.get_farm_info()
```
